# Staged rolout strategija #

Počela bih sa malim brojem korisnika koji bi se postepeno povećavao.
U tom periodu pratila bih stabilnost aplikacije, analitiku, kao i povratne informacije korisnika.
Izbegavala bih kraj radne nedelje kako bih imala dovoljno vremena da reagujem ukoliko dođe do problema.
Ukoliko bi se pojavio problem, kao što je crash koji pogađa određeni deo korisnika, zaustavila bih rollout dok se problem ne reši.
